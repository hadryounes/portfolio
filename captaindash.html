An exceptional mobile dashboard that reinvents reporting

The Project

Captain Dash is a tech startup based in Paris, building business management tools to help companies make better decisions based on their data. Captain Dash has an internationally renowned clientele including L’Oréal, Orange, SNCF, Universal Music, BNP Paribas...

Challenge:

Based on their growth strategy, Captain Dash decided to expand their target market by making their tools more accessible for small and medium businesses.


My mission:

Make the products of Captain Dash more usable by the self-service clients while keeping the excisting core technologies.


My tasks:

User Research, Information Architecture, Visual Design, Data Vizualisation


Tools:

Sketch, Invision, Zeplin


Project Status:

Ongoing.

User Research


After defining the project goals and outcomes, I expressed my advice of adopting a user-centred design process by starting from User Research. From this project perspective, the target user base is fully B2B, and they tend to have specific goals and needs.

The next step was to gather all the information that could help us define the user types. I conducted one-on-one and group meetings with different stakeholders (Head of Sales, Customer Success Managers) to get their inputs in order to build the company and user personas.

Deliverables:

  • Company Persona Profile
  • Persona Profile of the Decider
  • Persona Profile of the Administrator
  • Persona Profile of the End User

Information Architecture


Captain Dash’s products are defined by two segments, the first one is the Backend, which is a group of 2 platforms:

  • Staging Manager: Used to connect API as data sources and to create and manage reports;
  • Console: used to import and manage flat-files data (excel, csv, sql…).
The second segment is the Frontend, which is a group of applications used to preview the reports (desktop, tablet and mobile apps). This segment was out of scope for this current design project.

The first step of the Information Architecture was a thorough analysis of the current platforms to understand how they work and how they complete each other.

This diagram represents the distribution of the Staging Manager’s current interfaces and features. It is done to help find any possibilities to optimize the user journey and the navigation between the different parts of the application. The goal is to create a better experience that will be more optimized and that will make the use of the platform more accessible by the target users. The same process has been applied to the Console.

Deliverables:

  • Diagram of the current Staging Manager
  • Diagram of the current Console


Based on our users pain-points and needs, I decided to merge the Backend’s products in one application and give the users a one-stop-shop to import connect and manage their data, then to use it and create their reports that they can preview using the Frontend applications.

The following diagram is the result of the merge between the Manager and The Console, while keeping the features that are more adapted to the user journeys of self-service users.

Deliverables:

  • Diagram of the new application

User Interface Design


One of the most challenging aspects of this project is the need to keep the consistency of the current UI style guide, the decision has been made to limit the changes at the UI level, so I had to use the same UI elements in order to design better interfaces that will only need some minimal efforts from the engineering team.

I have started by building a styleguide on sketch using UI elements of the Staging Manager's interfaces. Then I used the elements to redesign the key screens.
